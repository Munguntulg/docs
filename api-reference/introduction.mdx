## Тойм

Энэ хэсэг нь таны API-ийн ишлэл (reference) болох бөгөөд дараах гол мэдээллийг агуулах ёстой:

- Base URL болон хувилбар (example: `https://api.example.com/v1`)
- Аутентификац (Bearer token, API key, OAuth) ба жишээ хүсэлт
- Гол endpoint бүлгүүд (Users, Orders, Products гэх мэт) ба тэдгээрийн үр дүнгийн схем
- Query/Path параметрүүд болон тэдгээрийн тайлбар
- Жишээ request/response (JSON), статус кодууд ба алдааны формат
- Pagination, rate limits, throttling талаархи заавар
- SDK/CLI эсвэл Postman коллекц-уудын холбоос
- OpenAPI спецификац (`openapi.json`) руу шууд холбоос

Бид энэ starter‑д OpenAPI файлыг ашиглаж байгаа тул бүх endpoint, схем, security тохиргоонуудыг [`openapi.json`](./openapi.json) файлд хадгална.

## Юу оруулах вэ (нарийвчилсан)

- **Base URL:** Тодорхой жишээ болон суурь зам (scheme, host, basePath).
- **Authentication:** хэрхэн Bearer token авах, request дээр хэрхэн оруулах (жишээ `Authorization: Bearer <token>`).
- **Endpoints:** бүлгээр ангилах, үр дүнгийн схем болон required/optional параметрүүд.
- **Request / Response жишээ:** curl болон JavaScript (fetch/axios) жишээнүүд.
- **Статус код ба алдаа:** 200, 201, 400, 401, 404, 500 гэх мэт болон error object-ын бүтэц.
- **Rate limits / Pagination:** хэр их хүсэлт/секунд, хэрхэн хуудаслах (limit/offset буюу cursor).
- **Хялбарчилсан жишээ:** Quick start — 1 минутанд token авч, нэг endpoint дуудах жишээ.
- **Reference Links:** OpenAPI файлын холбоос, SDK болон тест коллекцад хялбар хүрэх заавар.

## Authentication (жинхэнэ жишээ)

Бидний OpenAPI тохиргоо нь Bearer токенд тулгуурлана. Протоколын ерөнхий жишээ:

```http
GET /v1/plants HTTP/1.1
Host: api.example.com
Authorization: Bearer <YOUR_TOKEN>
Accept: application/json
```

OpenAPI-д security схем дараах байдлаар заагдсан байна. Илүү дэлгэрэнгүйг [`openapi.json`](./openapi.json) үзнэ үү.

```json
"security": [
  {
    "bearerAuth": []
  }
]
```


## Endpoint жишээнүүд ба лавлах

- `GET /v1/plants` — Бүтээгдэхүүний жагсаалт авах (query параметрыг дэмжиж болно)
- `POST /v1/plants` — Шинэ бичлэг үүсгэх (JSON body шаардлагатай)
- `DELETE /v1/plants/{id}` — Бичлэг устгах
- `POST /v1/webhook` — Вебхук хүлээн авах (жишээ)

Эдгээр жишээнүүдийг одоо OpenAPI спецификац дээр төвлөрүүлсэн тул тусдаа `endpoint` MDX файлууд устгагдсан. Илүү дэлгэрэнгүй request/response жишээг `openapi.json`-ээс үзнэ үү.
